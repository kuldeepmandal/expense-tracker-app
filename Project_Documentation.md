## 3. How Everything is Working (In Simple Words)
We are using the **MVC (Model-View-Controller)** pattern. This means the project code is cleanly split into three main parts so it's easy for teammates to manage:
- **Model (Data)**: The `app/models/` folder. These files handle talking directly to the database (fetching users, transactions, inserts, updates, and budgets).
- **View (Display)**: The `app/views/` folder. This contains all the HTML and CSS that the user interacts with on their screen (like the dashboard view, login page, and forms).
- **Controller (Logic)**: The `app/controllers/` folder. Think of this as the middleman! It receives requests from the user, asks the Model for needed data, and then passes that data to the View to be displayed.

**How URLs work:** All traffic passes through `index.php` first, which acts as a "Router" to direct the user to the correct controller based on the URL (e.g., going to `index.php?page=dashboard` triggers the DashboardController).

## 4. How Email Verification is Working
During registration, the system uses a popular, secure industry library called `PHPMailer` to send real emails to your users.
1. When a user registers, the `AuthController` successfully registers the user but also generates a random **6-digit OTP (One-Time Password)**.
2. The user's details are saved in the database, but their account access status (`is_verified`) is set to `0` (false/locked).
3. The `MailService` connects to an SMTP server (like Gmail) and securely sends an email to the user with their 6-digit OTP code attached.
4. The user is immediately redirected to the `/verify` page where they must enter the code from their email.
5. If the code matches the one stored in the database, their status permanently changes to `is_verified = 1` (true), and they are logged into the dashboard!

## 5. Use of All Main Files and Folders
Here is a simple directory map of what every major file does to help you navigate:
- `index.php`: The main entry point of the app. It manages routing the URLs to the appropriate controllers.
- `database_schema.sql`: Contains the predefined SQL commands to create the correct tables in your MySQL/PhpMyAdmin database.
- `composer.json` & `composer.lock`: Settings for managing extra PHP features (dependencies). In our case, used to install `PHPMailer`.
- `vendor/`: A folder automatically generated by Composer that contains the underlying `PHPMailer` library code.
- `app/config/MailService.php`: Contains the email sending logic and your SMTP server credentials.
- `app/controllers/...`: Contains the business logic rules (e.g., `AuthController.php` handles validating logins, `DashboardController.php` calculates the dashboard numbers).
- `app/models/...`: Contains the specific database queries (e.g., `User.php` touches the user table, `Transaction.php` touches the transactions table).
- `app/views/...`: Contains the frontend HTML code files (e.g., `dashboard.php`, `login.php`). `layout.php` is the master wrapper file containing the navigation sidebar so we don't have to rewrite the sidebar on every page.
